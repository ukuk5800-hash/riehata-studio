# 要件定義書

## サービス概要

多店舗スタジオ向けの予約・会員管理・体験申込・売上管理・マーケ分析を一元化したSaaS。
「ハコモノ（KARTE等）の代替」を目指し、スタジオ運営に特化した機能を提供する。

## 目的・背景

### 解決する課題

1. **スケジュール管理の手間** — 毎月クラスを作り直す作業をなくす
2. **予約開始日の手動管理** — ゴールド10日・一般15日などを自動制御
3. **講師変更の統制不足** — 講師が勝手に変更できる問題を防止
4. **ワークショップ運用コスト** — 告知・予約・決済を一気通貫に
5. **マーケティング不可視** — どこから来た人が入会したか分からない問題

### ターゲット

- 年商5億〜の多店舗スタジオ経営者・スタッフ
- 会員: 国内外問わず（多言語対応必須）

## 機能要件

### 運用思想（最重要）

- 管理者向け操作の主役は「チャット」
- フォームUIは補助（確認・微調整・監査用途）
- 自然文の頻出命令を高精度ルールベースで先に実装し、後からLLMへ置換できるアーキテクチャにする

### FR-01: 認証・権限管理

| No | 要件 | 優先度 |
|---|---|---|
| 1.1 | 5ロール (OWNER / STAFF / TEACHER / MEMBER / GUEST) | 必須 |
| 1.2 | メールアドレス認証（電話番号不要） | 必須 |
| 1.3 | Googleソーシャルログイン | 推奨 |
| 1.4 | パスワードリセット | 必須 |
| 1.5 | 講師は変更申請のみ。承認はOwner/Staff | 必須 |

### FR-02: 顧客向け予約

| No | 要件 | 優先度 |
|---|---|---|
| 2.1 | クラス一覧（フィルタ: 種別・曜日・講師） | 必須 |
| 2.2 | カレンダー表示 | 必須 |
| 2.3 | 会員種別による予約開始日出し分け | 必須 |
| 2.4 | 満席表示・キャンセル待ち登録 | 必須 |
| 2.5 | 体験レッスン申込（¥3,300） | 必須 |
| 2.6 | ワークショップ・単発イベント申込 | 必須 |
| 2.7 | 多言語切替（JA/EN） | 必須 |

| 3.2 | 予約開始日をプランごとに設定 | 必須 |
| 3.3 | 月間予約回数上限設定 | 必須 |
| 3.4 | 体験→入会キャッシュバック | 必須 |

| No | 要件 | 優先度 |
|---|---|---|
| 4.1 | 定期クラスのテンプレート（RecurringRule） | 必須 |
| 4.2 | 翌月分の自動生成（毎月25日） | 必須 |
| 4.3 | 代行講師設定 | 必須 |
| 4.4 | 振替クラス設定 | 必須 |

### FR-05: 変更申請フロー

| No | 要件 | 優先度 |
|---|---|---|
| 5.1 | 講師→申請（休講・日程変更・代行・時間変更） | 必須 |
| 5.2 | Owner/Staff→承認・却下・コメント | 必須 |
| 5.3 | 承認後に会員へ通知 | 必須 |

| No | 要件 | 優先度 |
|---|---|---|

| 6.2 | 体験申込・入会時に流入元紐付け | 必須 |
| 6.3 | 管理画面で流入元別件数・転換率表示 | 必須 |
| 6.4 | キャンペーン別成果表示 | 必須 |

### FR-07: AIアシスタント

| No | 要件 | 優先度 |
|---|---|---|
| 7.1 | チャットUI（管理画面内） | 必須 |
| 7.2 | 自然言語でのクラス操作指示（MVP: モック） | 必須 |
| 7.3 | Claude API接続（本番） | 推奨 |

## 非機能要件

| 区分 | 要件 |
|---|---|
| 多言語 | 日本語・英語（最低限） |
| 電話番号 | 一切不要 |
| レスポンシブ | スマートフォン・タブレット・PC対応 |
| 決済 | Stripe統合（MVP: モック） |
| セキュリティ | HTTPS必須・bcryptパスワードハッシュ・CSRFトークン |
| アクセシビリティ | WCAG 2.1 AA準拠目標 |
